# Documentation

This repo keeps docs grouped by intent so operational references, research notes, and execution plans are easy to find.

## Structure

- `docs/reference/`
  - stable external references and canonical source links
- `docs/research/`
  - exploratory analysis, API research, and methodology notes
- `docs/plans/`
  - execution tasklists and rollout plans
- `docs/product/`
  - UX/product design briefs and improvement proposals
